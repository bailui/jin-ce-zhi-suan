## 启动服务

开发环境（自动热重载）：

```bash
uvicorn quantify_api.main:app --host 0.0.0.0 --port 8000 --reload
```

生产环境（无热重载）：

```bash
uvicorn quantify_api.main:app --host 0.0.0.0 --port 8000
```

## 可写入的数据表

- dat_1mins
- dat_5mins
- dat_10mins
- dat_15mins
- dat_30mins
- dat_60mins
- dat_days

以上表支持统一 CRUD 接口：`/tables/{table}/rows`

## 新增分钟线表说明

本次新增并已支持增删改查的分钟线数据表为：

- dat_5mins
- dat_10mins
- dat_15mins
- dat_30mins
- dat_60mins

这些新增分钟线表与 `dat_1mins` 使用同一套 CRUD 路由，仅需替换 `{table}` 为对应表名即可。

## 通用 CRUD 接口

请求头统一携带：

- `x-api-key: <你的API密钥>`
- `Content-Type: application/json`（POST/PUT/DELETE 时）

### 1) 查询（Read）

GET `/tables/{table}/rows`

示例：

```bash
curl -X GET "http://127.0.0.1:8000/tables/dat_5mins/rows?limit=100&offset=0&order_by=trade_time&order_dir=asc&filter=code:eq:000001.SZ" ^
  -H "x-api-key: <你的API密钥>"
```

### 2) 新增（Create）

POST `/tables/{table}/rows`

示例：

```bash
curl -X POST "http://127.0.0.1:8000/tables/dat_10mins/rows" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"on_duplicate\":\"update\",\"rows\":[{\"code\":\"000001.SZ\",\"trade_time\":\"2026-03-17T10:00:00\",\"open\":10.1,\"high\":10.2,\"low\":10.0,\"close\":10.15,\"vol\":1000,\"amount\":101500,\"date\":\"2026-03-17\",\"pre_close\":10.0,\"change\":0.15,\"pct_chg\":1.5}]}"
```

### 3) 修改（Update）

PUT `/tables/{table}/rows`

示例：

```bash
curl -X PUT "http://127.0.0.1:8000/tables/dat_30mins/rows?limit=1" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"where\":{\"code\":\"000001.SZ\",\"trade_time\":\"2026-03-17T10:00:00\"},\"values\":{\"close\":10.18,\"high\":10.22}}"
```

### 4) 删除（Delete）

DELETE `/tables/{table}/rows`

示例：

```bash
curl -X DELETE "http://127.0.0.1:8000/tables/dat_60mins/rows?limit=1" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"where\":{\"code\":\"000001.SZ\",\"trade_time\":\"2026-03-17T10:00:00\"}}"
```

## K线缺失查询与补全接口

这组接口用于调用方做“缺失检测 + 自动补数”。

支持的 `table`：

- dat_1mins
- dat_5mins
- dat_10mins
- dat_15mins
- dat_30mins
- dat_60mins
- dat_days

说明：

- 分钟线表（`dat_1mins/dat_5mins/.../dat_60mins`）必须传 `start_time`、`end_time`
- 日线表（`dat_days`）必须传 `start_date`、`end_date`
- `code` 必须是 `000001.SZ` / `600000.SH` 这类格式

### 1) 查询缺失点位

GET `/kline/{table}/missing`

请求参数：

- `table`：路径参数，K线表名
- `code`：查询参数，股票代码
- `start_time`、`end_time`：分钟线区间（ISO8601）
- `start_date`、`end_date`：日线区间（YYYY-MM-DD）
- `limit`：最多返回的缺失点位明细数量（默认 5000）

分钟线示例：

```bash
curl -X GET "http://127.0.0.1:8000/kline/dat_5mins/missing?code=000001.SZ&start_time=2026-03-17T09:30:00&end_time=2026-03-17T15:00:00&limit=2000" ^
  -H "x-api-key: <你的API密钥>"
```

日线示例：

```bash
curl -X GET "http://127.0.0.1:8000/kline/dat_days/missing?code=000001.SZ&start_date=2026-03-01&end_date=2026-03-20&limit=500" ^
  -H "x-api-key: <你的API密钥>"
```

返回关键字段：

- `expected_points`：理论区间内应有点位数
- `existing_points`：目标表中已存在点位数
- `missing_points`：缺失点位总数
- `truncated`：是否因 `limit` 截断了明细
- `missing_trade_times` / `missing_dates`：缺失点位明细

### 2) 执行补全

POST `/kline/{table}/backfill`

请求体字段：

- `code`：股票代码
- `start_time`、`end_time`：分钟线补全区间（二选一组）
- `start_date`、`end_date`：日线补全区间（二选一组）
- `on_duplicate`：`error` / `ignore` / `update`（默认 `update`）
- `dry_run`：`true` 只预览不写入，`false` 执行写入
- `preview_limit`：返回预览缺失点数量上限（默认 200）

分钟线补全（先预览）：

```bash
curl -X POST "http://127.0.0.1:8000/kline/dat_30mins/backfill" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"code\":\"000001.SZ\",\"start_time\":\"2026-03-17T09:30:00\",\"end_time\":\"2026-03-17T15:00:00\",\"dry_run\":true,\"on_duplicate\":\"update\",\"preview_limit\":100}"
```

分钟线补全（实际写入）：

```bash
curl -X POST "http://127.0.0.1:8000/kline/dat_30mins/backfill" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"code\":\"000001.SZ\",\"start_time\":\"2026-03-17T09:30:00\",\"end_time\":\"2026-03-17T15:00:00\",\"dry_run\":false,\"on_duplicate\":\"update\"}"
```

日线补全（实际写入）：

```bash
curl -X POST "http://127.0.0.1:8000/kline/dat_days/backfill" ^
  -H "x-api-key: <你的API密钥>" ^
  -H "Content-Type: application/json" ^
  -d "{\"code\":\"000001.SZ\",\"start_date\":\"2026-03-01\",\"end_date\":\"2026-03-20\",\"dry_run\":false,\"on_duplicate\":\"update\"}"
```

返回关键字段：

- `missing_points`：待补全缺失点总数
- `preview_missing_points`：预览缺失点位（受 `preview_limit` 限制）
- `rowcount`：本次写入影响行数
- `dry_run`：是否仅预览

补全逻辑说明：

- 数据来源固定为 `dat_1mins`
- 会按目标周期聚合开高低收量额，并写入目标表
- 重复数据处理由 `on_duplicate` 控制
- `dat_1mins` 不支持自动补全（它是源数据表）
