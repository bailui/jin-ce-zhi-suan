
## 3. 首次启动前准备

项目已经提供一键启动脚本：

```bash
./start_local.sh
```

该脚本会自动完成以下工作：

- 检查并创建 `.venv` 虚拟环境
- 激活虚拟环境
- 自动读取本地私有配置文件 `config.private.json`
- 启动 Web 服务 `server.py`

如果脚本没有执行权限，可先运行：

```bash
chmod +x start_local.sh
```

## 4. 启动步骤

```bash
cd "/Users/bailu/Documents/New project/jin-ce-zhi-suan"

```

```text

```

## 5. 配置文件说明

### 主配置文件

项目主配置文件为：

```text
config.json
```

它保存系统默认配置，例如：

- 服务端口
- 风控参数
- 当前数据源类型
- 回测参数

### 私有配置文件

本地私有配置文件为：

```text
config.private.json
```

它用于保存不应提交到仓库的敏感配置，例如：

- `tushare_token`
- `default_api_key`
- `default_api_url`
- `llm_api_key`
- `strategy_llm_api_key`

当前模板如下：

```json
{
  "data_provider": {
    "tushare_token": "",
    "default_api_key": "",
    "default_api_url": "",
    "llm_api_key": "",
    "strategy_llm_api_key": ""
  }
}
```

## 6. 数据源配置建议

项目默认数据源是 `default`，但如果你没有作者私有接口，就需要改成其他可用方案。

推荐两种方式：

### 方式一：改成 `akshare`

适合本地快速体验，不需要额外 token。

在 `config.json` 中找到：

```json
"data_provider": {
  "source": "default"
}
```

改为：

```json
"data_provider": {
  "source": "akshare"
}
```

### 方式二：使用 `tushare`

如果你有 Tushare Token，可以在 `config.private.json` 中填写：

```json
{
  "data_provider": {
    "tushare_token": "你的token"
  }
}
```

然后把 `config.json` 里的：

```json
"source": "default"
```

改成：

```json
"source": "tushare"
```

## 8. 常见问题

### 1. 页面能打开，但回测拿不到数据

原因通常是数据源配置不可用。

优先检查：

- `config.json` 中的 `data_provider.source`
- `config.private.json` 中的 `tushare_token`
- 是否仍在使用作者私有 `default` 数据源

### 2. 提示私有配置文件缺失

如果你使用的是 `./start_local.sh`，脚本会自动创建 `config.private.json`。

如果手动启动，请确认项目根目录存在该文件。

### 3. 端口被占用

可改用其他端口，例如：

```bash
source .venv/bin/activate
python server.py --host 127.0.0.1 --port 8001
```

然后访问：

```text
http://127.0.0.1:8001
```

### 4. 缺少依赖

如果你没有用启动脚本，而是手动部署，先安装依赖：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt fastapi uvicorn tushare akshare
```

## 9. 推荐使用方式

建议你平时按下面流程使用：

1. 进入项目目录
2. 执行 `./start_local.sh`

4. 先把数据源切到 `akshare` 或填写 `tushare_token`
5. 再开始回测或其它操作
